# 공대표

파이널판타지14의 8인 공대원을 정해진 자리와 직업 우선순위에 맞춰 배치하고, 완성한 구성을 텍스트와 이미지로 공유하는 브라우저 도구입니다.

## 8인 고정 자리

공대표는 한 개의 8인 파티를 다음 자리로 구성합니다.

- `MT` — 메인 탱커
- `ST` — 서브 탱커
- `MH` — 메인 힐러
- `SH` — 서브 힐러
- `D1` — 근딜
- `D2` — 근딜
- `D3` — 유격대
- `D4` — 캐스터

파티 수나 정원을 따로 설정할 필요 없이 여덟 자리를 채우는 데 집중할 수 있습니다.

## 사용 흐름

1. 빈 구성표에서 공대 이름과 일정을 입력합니다.
2. 공대원의 닉네임과 주직업을 등록하고, 가능한 부직업이 있다면 하나 더 추가합니다.
3. `추천 배치`를 누르면 전체 조합에서 주직업 배치를 최대한 유지하고, 꼭 필요한 자리에만 부직업을 사용합니다.
4. 필요하면 공대원을 직접 선택하거나 끌어 원하는 자리로 옮깁니다.
5. 완성한 결과를 디스코드용 텍스트로 복사하거나 고해상도 PNG로 복사·저장합니다.

처음 열었을 때는 모든 자리가 비어 있는 8인 구성표가 표시됩니다. 입력한 내용은 현재 브라우저에 자동으로 저장됩니다.

## 직업 우선순위

- 주직업은 추천 배치에서 가장 먼저 사용합니다.
- 부직업은 주직업만으로 자리를 채울 수 없을 때 사용합니다.
- 각 직업은 배치할 수 있는 자리와 연결되며, 맞지 않는 자리에는 자동으로 들어가지 않습니다.
- 가능한 조합이 여러 개라면 주직업 배치를 최대한 유지한 상태에서 빈자리를 채웁니다.
- 자동 배치 뒤에도 모든 자리를 직접 조정할 수 있습니다.

## 저장과 공유

공대 이름, 일정, 명단, 직업 우선순위와 배치는 브라우저의 로컬 저장소에 자동으로 저장됩니다. 서버로 전송되지 않으며 다른 브라우저나 기기와 자동으로 동기화되지 않습니다.

HTTPS 환경에서 브라우저가 이미지 클립보드를 지원하면 PNG를 바로 복사합니다. 지원하지 않는 환경에서는 별도의 `PNG 저장` 버튼을 이용할 수 있습니다.

## 실행과 테스트

별도 빌드 과정 없이 `index.html`을 열거나 저장소 루트에서 정적 서버를 실행합니다.

```bash
python3 -m http.server 8000
node --test raid-maker/tests/*.test.js
```

GitHub Pages에서는 `/raid-maker/` 경로로 접근할 수 있습니다.
