# 언제표

월요일부터 일요일까지 시간별 가능 여부를 칠하고 링크·텍스트·이미지로 공유하는 브라우저 기반 주간 일정표 도구입니다.

## 기능

- 월–일 × 24시간, 한 시간 단위 선택
- 하루 시작 시각(기본 08:00)과 표의 시작 요일(기본 월요일) 설정
- 클릭·터치와 드래그로 선택 또는 선택 해제
- 요일·시간 머리글 전체 선택, 실행 취소, 일정표 초기화
- 선택 상태를 21바이트로 압축한 공유 링크
- 연속 시간을 합친 요일별 텍스트 복사
- 고해상도 PNG 복사와 저장
- 공유받은 일정 자동 보관과 저장 목록 검색·불러오기·수정·삭제
- 저장 목록에서 여러 일정을 골라 바로 취합
- 링크 하나에서 최대 8명이 직접 입력하는 Firebase 온라인 취합방
- 온라인 참여자 실시간 갱신, 방 잠금, 방장 응답 정리와 방 삭제
- 여러 공유 링크의 가능 인원을 칸별로 합산하고 참여자 이름 확인
- 취합 이미지를 전체 현황·전원 가능 시간·직접 고른 시간 범위로 PNG 복사하거나 저장
- 취합 결과를 이름 붙여 보관하고 새 일정만 골라 추가하거나 링크로 공유
- 키보드 방향키 탐색 및 Space/Enter 선택
- 브라우저 로컬 처리와 `IndexedDB` 자동 저장·파일 보관함 백업

일정 이름 입력칸은 처음에 비어 있습니다. 이름 없이 공유 링크를 복사하면 먼저 이름을 입력하도록 안내합니다. `초기화` 버튼은 선택 시간과 일정 이름을 지우고 하루 시작을 08:00, 시작 요일을 월요일, 시간대를 현재 브라우저 기준으로 되돌립니다.

## 하루 시작 시각과 시작 요일

기본 시작 시각은 08:00입니다. 이때 각 요일 열의 행은 `08:00–23:00`, `익일 00:00–07:00` 순서로 이어집니다. 예를 들어 월요일 열에서 23시, 0시, 1시를 선택하면 월요일의 `23:00–익일 02:00` 한 구간으로 복사됩니다. 하루 전체를 선택하면 `08:00–익일 08:00`으로 표시됩니다.

시작 시각은 0시부터 23시까지 바꿀 수 있으며 공유 링크의 `h` 값에 함께 저장됩니다. 시작 요일을 바꾸면 선택 데이터는 그대로 둔 채 열 순서만 회전합니다. 예를 들어 토요일 시작은 `토·일·월·화·수·목·금` 순서로 표시되며 텍스트와 PNG도 같은 순서를 사용합니다.

시작 요일은 공유 링크의 `d` 값에 저장됩니다. `h`와 `d`가 없는 기존 공유 링크는 이전 동작과 같은 00:00·월요일 시작 일정으로 불러옵니다.

## 온라인 취합방

`/schedule-maker/room.html`에서는 방장이 링크 하나를 만든 뒤 최대 8명이 같은 페이지에서 닉네임과 가능한 시간을 직접 저장할 수 있습니다. 한 사람이 저장하거나 수정하면 참여자 목록과 겹침표가 Firestore 실시간 구독으로 갱신됩니다.

온라인 응답은 방 문서 하나의 `responses` map에 저장됩니다. 각 응답의 168칸 선택 데이터는 기존 링크와 같은 21바이트·28자 base64url 문자열을 재사용합니다. Firestore Security Rules는 다음 권한을 서버에서 검사합니다.

- 전체 방 목록 조회 차단
- 참여자 최대 8명
- 각 익명 사용자는 자기 UID의 응답만 수정
- 잠긴 방에서는 참여자 수정 차단
- 방장만 입력 마감·다른 응답 삭제·방 삭제

참여자는 로그인 화면 없이 Firebase 익명 인증을 사용합니다. 익명 UID와 방장 권한은 해당 브라우저에 연결되므로 브라우저 데이터를 지우거나 다른 기기로 옮기면 기존 응답의 수정 권한을 잃을 수 있습니다. 방 링크를 아는 사람은 방의 닉네임과 취합 결과를 볼 수 있습니다.

Firebase 공개 웹 설정, 익명 Authentication, Firestore Rules를 연결하는 순서는 [Firebase 연결 안내](./FIREBASE_SETUP.md)를 참고합니다. 서비스 계정 키는 사용하지 않습니다. 기존 일정 작성·공유 링크·수동 취합·로컬 보관함은 Firebase와 관계없이 계속 동작합니다.

## 여럿 합쳐보기

여러 사람이 보낸 언제표 링크를 한 줄에 하나씩 붙여넣으면 각 시간 칸에 가능한 인원수를 표시합니다. 겹침 색은 1명부터 8명 이상까지 8단계로 진해집니다. 칸을 가리키거나 탭하면 가능한 사람과 불가능한 사람의 일정 이름을 확인할 수 있습니다. 서로 다른 하루 시작 시각은 실제 요일에 맞춰 자동 정렬하며, 결과의 첫 요일은 별도로 바꿀 수 있습니다.

취합 화면은 `/schedule-maker/compare.html`에 분리되어 있습니다. 저장 목록에서 여러 일정을 체크한 뒤 `선택 일정 취합하기`를 누르면 링크를 다시 붙여넣지 않아도 곧바로 취합됩니다.

취합 결과의 `이미지 복사`와 `PNG 저장`에서는 `전체 현황`, `전원 가능만`, `직접 선택` 가운데 이미지 범위를 고를 수 있습니다. 직접 선택 모드에서는 취합표의 칸을 클릭하거나 키보드 Space/Enter로 고른 시간만 남깁니다. 이미지 위아래의 빈 시간 행은 자동으로 잘라내되 첫 시간과 마지막 시간 사이의 빈 행은 시간 흐름을 위해 유지합니다. 이미지 클립보드를 지원하지 않는 브라우저에서는 `이미지 복사`도 PNG 저장으로 전환됩니다.

취합한 참여자와 결과의 시작 시각·요일은 이름을 붙여 별도의 취합표로 저장할 수 있습니다. 저장한 취합표는 참여 일정의 스냅샷을 가지므로 나중에 개별 일정 저장본을 삭제해도 그대로 유지됩니다. 취합 링크는 `취합 링크 복사`를 누를 때만 만들어지며, 링크를 받은 사람은 같은 참여자와 보기 설정을 불러온 뒤 자기 브라우저 보관함에 자동 저장합니다. 불러온 뒤에는 주소창의 긴 공유 데이터가 바로 제거됩니다.

반복 주간표만으로는 서로 다른 시간대를 정확히 변환할 수 없으므로, 첫 번째 링크와 시간대가 다른 일정은 경고와 함께 집계에서 제외합니다. 일정 이름에 닉네임을 넣어 공유하면 참여자 이름으로 사용됩니다.

## 저장한 일정

공유 링크로 연 일정은 내용을 읽은 직후 공용 `IndexedDB` 보관함에 자동 저장하고, 저장 완료를 확인한 뒤 곧바로 개별 일정 목록으로 이동합니다. 새로 받은 일정은 목록 맨 위에서 확인할 수 있습니다. 공유 링크를 만든 내 일정도 저장 목록에 함께 보관됩니다. `/schedule-maker/saved.html`에서 다음 작업을 할 수 있습니다.

- 이름이나 시간대로 검색
- 저장본을 사본으로 불러오기
- 저장본의 시간표와 이름 수정
- 필요 없는 일정 삭제
- 여러 일정을 선택해 바로 취합
- 선택한 일정만 기존 취합표에 추가하고 정확한 중복은 제외
- 저장한 취합표 열기·이름 수정·삭제·공유 링크 복사

기본 작업본은 같은 브라우저와 사이트 출처에서만 사용할 수 있습니다. 도구함 첫 화면에서 파일 보관함을 연결하면 일정 초안·개별 일정 목록·저장한 취합표도 함께 백업할 수 있습니다. 파일을 다른 기기에서 열거나 휴대용 텍스트로 옮기기 전까지는 자동으로 기기 간 동기화되지 않습니다.

## 실행과 테스트

별도 빌드 과정 없이 `index.html`을 열거나 저장소 루트에서 정적 서버를 실행합니다.

```bash
python3 -m http.server 8000
node --test schedule-maker/tests/*.test.js shared/tests/*.test.js
```

GitHub Pages에서는 일정 작성 `/schedule-maker/`, 온라인 방 `/schedule-maker/room.html`, 수동 취합 `/schedule-maker/compare.html`, 저장 목록 `/schedule-maker/saved.html` 경로로 접근할 수 있습니다.
