# 작은 도구함

설치 없이 브라우저에서 바로 사용하는 작은 웹 도구들을 모아두는 GitHub Pages 저장소입니다.

## 도구

- [반듯표](./table-maker/) — 한글·전각 문자를 지원하는 아스키(ASCII) 테이블 생성기
- [언제표](./schedule-maker/) — 가능한 시간을 공유하고, 링크 하나의 온라인 방에서 최대 8명이 직접 입력·취합하는 주간 일정표
- [언제표 취합](./schedule-maker/compare.html) — 여러 사람의 언제표 링크에서 겹치는 시간을 찾는 도구
- [언제표 온라인 방](./schedule-maker/room.html) — Firebase에 각자 일정을 저장하고 실시간으로 겹치는 시간을 확인하는 취합방
- [꾸준표](./habit-maker/) — 목표량과 실천 정도를 기록하는 월간 습관 캘린더
- [공대표](./raid-maker/) — 주직업과 부직업 우선순위로 파이널판타지14 8인 공대를 구성하는 도구
- [하루기록](./daily-log/) — 식사·음주·컨디션과 메모를 날짜별로 남기는 생활 기록 달력

## 새 도구 추가

1. 저장소 루트에 새 도구 폴더를 만듭니다.
2. 폴더 안에 독립적으로 실행되는 `index.html`을 둡니다.
3. 루트 `index.html`의 `.tool-grid`에 해당 도구 카드를 추가합니다.
4. 새 페이지의 canonical과 공유 메타데이터를 추가하고 `sitemap.xml`에 공개 주소를 등록합니다.

```text
/
├── index.html
├── hub.css
├── hub-vault.js
├── sitemap.xml
├── shared/
├── table-maker/
├── schedule-maker/
├── habit-maker/
├── raid-maker/
├── daily-log/
└── next-tool/
```

GitHub Pages의 배포 원본은 `main` 브랜치의 `/ (root)`로 설정합니다.

## Google 검색 등록

공개 검색 페이지는 `https://huis-snow.github.io/tools/` 아래의 canonical 주소를 사용하며, 검색 허용 페이지 목록은 루트 [`sitemap.xml`](./sitemap.xml)에서 관리합니다. 브라우저 로컬 데이터만 보여주는 일정 보관함은 `noindex, follow`로 검색 대상에서 제외합니다.

Google Search Console에는 URL 접두어 속성 `https://huis-snow.github.io/tools/`을 등록합니다. 계정별 HTML 확인 태그를 루트 `index.html`의 `<head>`에 추가해 소유권을 확인한 뒤 `https://huis-snow.github.io/tools/sitemap.xml`을 제출하고, 새로 배포한 핵심 페이지는 URL 검사에서 색인을 요청합니다.

## 기록 보관함

기록을 저장하는 도구는 공용 보관함을 사용합니다. 작업 중인 데이터는 이 사이트 출처의 `IndexedDB`에 먼저 저장되므로 GitHub Pages 같은 정적 사이트에서도 별도 서버 없이 동작합니다. 기존 `localStorage` 기록은 각 도구를 처음 열 때 성공 여부를 확인한 뒤 자동으로 이전합니다.

언제표의 온라인 취합방은 예외적으로 방 이름·닉네임·선택 시간을 Firebase Cloud Firestore에 저장합니다. 온라인 방 화면에서 이 차이를 별도로 안내하며, 로컬 초안과 기존 수동 링크 취합은 계속 브라우저 안에서만 처리됩니다.

루트 첫 화면에서 보관함 JSON 파일을 만들거나 기존 파일을 열 수 있습니다. Chromium 계열 브라우저에서는 사용자가 허용한 파일에 변경분을 주기적으로 반영하고 최근 파일을 다시 연결할 수 있습니다. 직접 파일 쓰기를 지원하지 않는 브라우저에서는 파일 불러오기와 새 JSON 다운로드 방식으로 대체합니다. 체크섬은 파일 손상을 찾기 위한 것이며 암호화는 아닙니다. 브라우저 보안상 전체 파일 경로는 알 수 없고 파일명만 표시합니다.

기록 도구의 상태 배지는 브라우저 작업본 저장과 연결 파일 동기화를 구분해 표시합니다. 앱 데이터가 JSON·스키마 검증을 통과하지 못하면 원문을 복구 키에 격리하고 자동 저장을 멈춰 빈 데이터로 덮어쓰지 않습니다. 불러오기·초기화·삭제 같은 위험 작업 전에는 현재 보관함 전체를 복원 지점으로 만들며, 최근 5개를 루트 보관함 패널에서 되돌리거나 삭제할 수 있습니다. 복원 지점은 현재 브라우저에만 보관되고 JSON·`TOOLS1.` 백업에는 포함되지 않습니다.

네이버 웨일은 저장해 둔 파일 권한을 다시 요청하는 과정에서 브라우저가 종료될 수 있고, 저장 파일 선택기로 기존 파일을 다시 고르면 내용이 먼저 비워질 수 있습니다. 현재는 직접 파일 연결을 사용하지 않고 JSON 파일 불러오기·다운로드 안전 모드로 자동 전환합니다. IndexedDB 작업본과 각 도구의 기록은 그대로 유지됩니다.

파일 없이도 현재 브라우저에서 계속 사용할 수 있지만, 브라우저 데이터 삭제나 저장 공간 정리로 기록이 사라질 수 있으므로 중요한 기록은 JSON 파일이나 `TOOLS1.` 휴대용 텍스트로 백업하세요. 새 기록 도구를 연결하는 방법은 [공용 보관함 안내](./shared/README.md)를 참고합니다.
