# 작은 도구함

설치 없이 브라우저에서 바로 사용하는 작은 웹 도구들을 모아두는 GitHub Pages 저장소입니다.

## 도구

- [반듯표](./table-maker/) — 한글·전각 문자를 지원하는 ASCII 테이블 생성기
- [언제표](./schedule-maker/) — 가능한 시간을 공유하고, 받은 일정을 저장·수정·취합하는 주간 일정표
- [언제표 취합](./schedule-maker/compare.html) — 여러 사람의 언제표 링크에서 겹치는 시간을 찾는 도구
- [꾸준표](./habit-maker/) — 목표량과 실천 정도를 기록하는 월간 습관 캘린더
- [공대표](./raid-maker/) — 주직업과 부직업 우선순위로 파이널판타지14 8인 공대를 구성하는 도구
- [하루기록](./daily-log/) — 식사·음주·컨디션과 메모를 날짜별로 남기는 생활 기록 달력

## 새 도구 추가

1. 저장소 루트에 새 도구 폴더를 만듭니다.
2. 폴더 안에 독립적으로 실행되는 `index.html`을 둡니다.
3. 루트 `index.html`의 `.tool-grid`에 해당 도구 카드를 추가합니다.

```text
/
├── index.html
├── hub.css
├── hub-vault.js
├── shared/
├── .nojekyll
├── table-maker/
├── schedule-maker/
├── habit-maker/
├── raid-maker/
├── daily-log/
└── next-tool/
```

GitHub Pages의 배포 원본은 `main` 브랜치의 `/ (root)`로 설정합니다.

## 기록 보관함

기록을 저장하는 도구는 공용 보관함을 사용합니다. 작업 중인 데이터는 이 사이트 출처의 `IndexedDB`에 먼저 저장되므로 GitHub Pages 같은 정적 사이트에서도 별도 서버 없이 동작합니다. 기존 `localStorage` 기록은 각 도구를 처음 열 때 성공 여부를 확인한 뒤 자동으로 이전합니다.

루트 첫 화면에서 보관함 JSON 파일을 만들거나 기존 파일을 열 수 있습니다. Chromium 계열 브라우저에서는 사용자가 허용한 파일에 변경분을 주기적으로 반영하고 최근 파일을 다시 연결할 수 있습니다. 직접 파일 쓰기를 지원하지 않는 브라우저에서는 파일 불러오기와 새 JSON 다운로드 방식으로 대체합니다. 체크섬은 파일 손상을 찾기 위한 것이며 암호화는 아닙니다. 브라우저 보안상 전체 파일 경로는 알 수 없고 파일명만 표시합니다.

네이버 웨일은 저장해 둔 파일 권한을 다시 요청하는 과정에서 브라우저가 종료될 수 있는 호환성 문제가 확인되어, 현재는 직접 재연결을 사용하지 않고 JSON 파일 불러오기·다운로드 안전 모드로 자동 전환합니다. IndexedDB 작업본과 각 도구의 기록은 그대로 유지됩니다.

파일 없이도 현재 브라우저에서 계속 사용할 수 있지만, 브라우저 데이터 삭제나 저장 공간 정리로 기록이 사라질 수 있으므로 중요한 기록은 JSON 파일이나 `TOOLS1.` 휴대용 텍스트로 백업하세요. 새 기록 도구를 연결하는 방법은 [공용 보관함 안내](./shared/README.md)를 참고합니다.
